Overview

This is an on-going group project. I teamed up with two other developers. My role is to research, provide a project brief, create wireframes and design the final prototype.


Our goal is to create a hair salon searching and booking system for desktop and mobile experience. Our aim is to provide people a smoother and more organised experience when booking their next hair appointment. Our product will be a great communication platform for both customer and service provider to have follow-ups after the appointment and to preserve the service histories.

Requirment

  1. Product is responsive to allow for good customer experience on both desktop and mobile devices.
  2. User (customer) is able to search hair salon by name, location and date/time.
  3. User (customer) is able to book an appointment through website/app.
  4. User (salon) is able to recieve and approve appointment request in the website/app.
  5. User is able to keep track of appointment history .
  6. User (customer) is able to search hair style and post pictures of own.
  7. User (salon) is able to schedule staff and organize appointment slots in the website/app.
  8. Communication between users through the website/app.

Sign up page

First, I designed each box with a label, then I added pre-filled text in the box. Having both a label and pre-filled text gave crowded looks. I removed labels and was much cleaner but then it made me concern about accessibility. How would a screen reader user experience this? What about a user with a cognitive disability? Would they be confused about what was each box was for after started typing and not able to see the pre-filled text anymore?


I needed to have another indicator besides pre-filled text to increase the clarity of indication for each box. My solution was to have icons. The icon can indicate what each box is for in simple form with minimum space.
